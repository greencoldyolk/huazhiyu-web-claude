# 华之喻官网首页交互与视觉实现规格

> 适用对象：Codex / 前端实现 Agent  
> 版本：v0.2  
> 目标：在已确认的 6 个首页 section 基础上，实现“现代技术公司 × 专业服务机构 × 编辑式品牌官网”的高质感页面。  
> 原则：静态状态先成立，动画只负责增强；琥珀金只做小面积反色点缀，不允许大面积泛黄。

---

# 0. 实现优先级

## P0：必须完成

- 页面结构、排版、响应式布局
- 全站颜色 token
- Header 滚动态
- Hero 标题、按钮、抽象结构视觉

## 1.2 琥珀金边界规则

### 允许

- 小型节点、编号、圆点、箭头端点
- 某个重点卡片的局部暖白底
- 卡片边缘一圈极弱暖色光
- CTA 按钮高光
- 深色区右下角弧光
- 上传区 hover 的一次性扫光
- 时间线当前节点

### 禁止

- 整个 section 使用黄色背景
- 一整张卡使用明显黄底
- 多张服务卡同时发黄
- 大面积橙色按钮
- 黄色文字作为正文主色
- 黄色和蓝紫同时高饱和并列

### 面积限制

- 单屏琥珀金视觉面积不超过约 5%
- 单张卡片暖色区域不超过约 20%
- 整页只有 1 张服务卡允许带明显暖调

# 2. 全局通用状态

## 默认态

- 所有 section 静态截图必须成立
- 不依赖 hover 才能理解可点击性
- 边框统一使用 `--line`
- 圆角统一使用 10 / 16 / 24 三档
- 阴影不超过三档

## Hover 态

- 位移限制在 `translateY(-3px)` 至 `-4px`
- 箭头横向移动 3–5px
- 边框变亮或出现微弱光晕
- 不允许明显放大
- 不允许持续闪烁或循环动画

## Focus-visible 态

```css
:focus-visible {
  outline: 3px solid rgba(91, 92, 226, 0.28);
  outline-offset: 3px;
}
```

## Disabled 态

- `opacity: .48`
- `cursor: not-allowed`
- 不允许 hover 位移

## Reduced motion

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

# 3. Header

## 默认态

- 高度：72–76px
- 背景：透明白或 `rgba(255,255,255,.72)`
- Logo 左对齐，桌面宽度 260–285px
- 导航居中偏右
- 右侧保留一个“提交新事项”主 CTA
- 无明显阴影
- 底部分隔线默认极弱或不显示

## Scroll 后状态

- `position: sticky; top: 0; z-index: 100`
- 背景变为 `rgba(255,255,255,.88)`
- `backdrop-filter: blur(18px) saturate(140%)`
- 底部出现 1px 渐变线
- Logo 缩小不超过 4%
- Header 高度可以从 76px 缩至 68px

## Hover 态

### 导航项

- 文字颜色从 `--text` 变 `--indigo-800`
- 下方 1px 细线从中心向两侧展开
- 时长 180–220ms

### Header CTA

- 上移 1px
- 背景亮度提升 3%–5%
- 内部高光轻微移动
- 箭头右移 4px

## 独有变化

- Logo 中现有黄色三角保留，但重绘为柔和琥珀金
- 不增加其他大面积金色

# 4. Hero

## 默认态

### 左侧文案

- H1 两行以内
- 字号：桌面 64–72px，移动端 40–48px
- 关键词“创新价值”使用蓝紫渐变
- 正文 1–2 行
- 主按钮 + 低强调文字链接
- 底部 3 个轻量利益点

### 右侧抽象视觉

由以下层组成：

1. 低透明度大面积 radial glow
2. 3–5 层半透明矩形/平面
3. 极细路径线
4. 2–4 个节点
5. 1 个琥珀金节点
6. 1 个琥珀金底座或端点
7. 很弱的透视网格

## Hover / Pointer move 态

- 整组视觉跟随鼠标移动 4–8px
- 前景层和背景层使用不同位移量形成轻微视差
- 不做明显 3D 翻转
- 节点可轻微放大至 `1.03`
- 琥珀节点亮度轻微增强

## CTA Hover

- 主按钮上移 2px
- 背景渐变轻微向右移动
- 箭头右移 4px
- 出现 1px 内高光

## 独有变化

- 琥珀金仅出现在 1 个节点和 1 个底座
- 不允许 Hero 右侧出现第二套暖色渐变
- 抽象视觉必须能由 CSS + SVG 实现，不依赖位图

## 移动端

- 右侧视觉缩为顶部或底部辅助图层
- 视觉高度约 220–260px
- 取消鼠标跟随
- 保留静态层叠和一个琥珀节点

# 5. 核心服务入口

## 布局

- 桌面：3 列 × 2 行
- 平板：2 列 × 3 行
- 移动：1 列 × 6 行
- gap：24px 桌面，16px 移动
- 卡片高：180–210px 桌面，136–156px 移动

## 普通卡默认态

- 白色或冷白半透明底
- 1px `--line` 边框
- `box-shadow: var(--shadow-sm)`
- 左上角小号编号
- 标题、短说明、右下箭头
- 背景可带非常弱的编号水印或几何切角

## 普通卡 Hover 态

- `transform: translateY(-3px)`
- `box-shadow: var(--shadow-md)`
- 边框变为 `rgba(91,92,226,.22)`
- 底部细线从 24px 延伸至 80px
- 箭头右移 4px
- 背景几何层透明度增加约 8%

## 哪张卡略有不同

### “专利申请”重点卡

#### 默认态

- 使用暖白底，不使用明显黄色底
- 建议背景：

```css
background:
  radial-gradient(circle at 82% 24%, rgba(243,196,107,.16), transparent 30%),
  linear-gradient(135deg, #fffdfa 0%, #fffaf2 52%, #ffffff 100%);
```

- 边框为 `rgba(230,169,61,.28)`
- 右上角或右侧可有一个小型琥珀线框图章
- 左上编号可以使用 `--amber-500`
- 卡片主体文字仍用深靛蓝
- 琥珀视觉面积不超过卡片 20%

#### Hover 态

- 上移 4px
- 暖色 radial glow 增强，但不扩散到整张卡
- 边框变为 `rgba(230,169,61,.50)`
- 小型图章轻微旋转 2° 或缩放到 1.04
- 底部暖色细线由左向右展开
- 允许一次性暖色高光扫过，时长 650–750ms

#### 禁止

- 整张卡变成黄卡
- 使用高饱和橙色图标
- 使用金色正文
- 让该卡比其他卡高出超过 4px
- 持续发光

## 其他卡差异

- “知识产权咨询”可用极浅灰蓝紫底
- 其余卡保持白/冷白
- 只有 1 张暖调卡，最多再有 1 张冷紫调卡

# 6. Section 过渡

## 默认态

- 不使用硬切色
- Hero → 服务：使用浅色弧形或波形过渡
- 服务 → 为什么选择：背景从浅灰蓝渐变回白色
- 为什么选择 → 专业依据：直接进入深色，但边缘可用细线或 1px 光带
- 专业依据 → 代表性工作：回到白色
- 提交新事项 → Footer：深色自然衔接

## Hover 态

- 无 hover

## 滚动进入态

- Section 内容上移 12–20px 淡入
- stagger 50–80ms
- 背景本身不做大幅移动

# 7. 为什么选择华之喻

## 默认态

- 左侧：Section 标题 + 一句定位文案
- 右侧：3 条编辑式优势列表
- 不使用卡片背景
- 每条有序号、标题、说明、右侧轻箭头
- 条目之间使用水平分隔线

## Hover 态

- 当前条目标题变深
- 背景出现非常弱的横向 glow
- 序号从描边/浅色变成实色
- 左侧竖线或圆点移动到当前条目
- 当前条目箭头右移 4px
- 其他条目透明度降到 0.72–0.82

## 独有变化

- 桌面左侧可 `position: sticky; top: 140px`
- sticky 距离控制在 200–300px
- 不做长篇滚动叙事
- 第 1 条默认激活

## 移动端

- 取消 sticky
- 三条纵向堆叠
- 每条触摸时显示 active 状态

# 8. 专业依据

## 默认态

- 深墨靛蓝满宽背景
- 3 个真实依据列
- 每列大数字/事实 + 短说明
- 背景带极弱噪点、径向光、细网格
- 列之间用低亮度发光竖线分隔
- 琥珀金只在右下角弧光或一个节点上出现

## Hover 态

- 当前列背景亮度提升 2%–3%
- 数字轻微放大到 1.015
- 图标描边亮度增强
- 分隔线临近区域轻微发光

## Scroll 进入态

- 数字 count-up 一次
- 图标与说明 stagger 淡入
- 动画完成后保持静止

## 独有变化

- 此 section 是全页最强反色区之一
- 不允许再次叠加大面积紫色卡片
- 不允许出现 3 张独立玻璃卡

# 9. 代表性工作

## 默认态

### 重点案例

- 左侧大展板
- 包含：大号编号、类别小标签、标题、短说明、详情箭头
- 背景使用冷蓝灰层次和轻微网格
- 可有局部裁切的大号数字

### 次级案例

- 右侧 2 条文字项目
- 无独立厚重卡片
- 使用轻容器或分隔线
- 可有编号 02 / 03

### 时间线节点

- 中间细竖线
- 当前节点使用小琥珀圆点
- 其他节点使用冷灰蓝

## 重点案例 Hover 态

- 整体上移 3px
- 背景纹理慢速位移 6–10px
- 大号编号透明度增强
- 详情箭头右移 4px
- 边缘出现极弱蓝紫光

## 次级案例 Hover 态

- 标题变深
- 背景轻微亮起
- 箭头右移
- 左侧时间线对应节点从冷灰变为琥珀金

## 哪个元素略有不同

- 只有当前时间线节点使用琥珀金
- 重点案例本体保持冷色，不做暖色大卡

# 10. 提交新事项

## Section 默认态

- 深墨靛蓝到灰蓝紫渐变背景
- 右下角加入柔和琥珀金光晕
- 背景包含极弱波纹/路径线
- 左侧说明，右侧表单
- 表单为半透明浅色玻璃容器

## 表单容器默认态

- `background: rgba(255,255,255,.14)` 或浅色方案中 `rgba(255,255,255,.92)`
- `backdrop-filter: blur(18px)`
- 1px 半透明白边
- 内部控件之间 16–20px gap

## 输入框默认态

- 高度 50–54px
- 半透明白底
- 1px `rgba(255,255,255,.22)` 或浅色 `--line`
- placeholder 使用低对比灰

## 输入框 Hover 态

- 边框轻微变亮
- 背景透明度增加

## 输入框 Focus 态

- focus ring 使用蓝紫，不用琥珀金
- 阴影：`0 0 0 4px rgba(91,92,226,.14)`
- label/边框亮度增强

## 上传区默认态

- 虚线边框
- 上传图标
- 支持 Word / PDF 等说明
- 不使用聊天气泡或 AI 输入框造型

## 上传区 Hover 态

- 边框从虚线冷灰变蓝紫
- 上传图标上移 2px
- 一道琥珀高光从左向右扫过一次
- 背景出现轻微暖色 radial glow

## 上传区 Drag-over 态

- 边框变实线或高亮虚线
- 背景亮度提高
- 文案替换为“释放以上传”
- 琥珀高光更明显，但不持续闪烁

## 上传完成态

- 显示文件名、大小、删除按钮
- 文件行淡入 + 轻微上移
- 成功图标使用蓝紫或冷色，不用大面积绿色

## 提交按钮默认态

- 使用暖金渐变作为此 section 的主要琥珀使用点
- 按钮高度 52–56px
- 文字深靛蓝
- 右侧箭头

## 提交按钮 Hover 态

- 上移 2px
- 渐变高光横向移动
- 阴影增强
- 箭头右移 4px

## 提交按钮 Loading 态

- 文案变“正在提交”
- 显示小型 spinner
- 禁止重复点击
- 不改变按钮尺寸

## 提交成功态

- 表单容器切换为成功确认
- 使用蓝紫成功图形 + 小型琥珀点缀
- 提供“继续提交”轻按钮

## 独有变化

- 这是全页琥珀金面积最大的地方
- 但琥珀只集中在按钮、右下角光晕和上传 hover
- 背景主体仍为墨靛蓝 + 灰蓝紫

# 12. Section reveal 统一规则

## 默认态

```css
.reveal {
  opacity: 0;
  transform: translateY(18px);
}

.reveal.is-visible {
  opacity: 1;
  transform: translateY(0);
  transition:
    opacity 520ms var(--ease-emphasized),
    transform 520ms var(--ease-emphasized);
}
```

## 卡片 stagger

- 每张卡延迟 50–80ms
- 最大延迟不超过 400ms
- 一次播放后不重复

## 禁止

- 从左右大幅飞入
- 缩放从 0 开始
- 连续重复 reveal
- 页面滚动时反复闪现

# 13. 响应式规则

## Desktop ≥ 1200px

- 1240px 最大内容宽度
- 12 栏 grid
- Hero 6/6 或 5/7
- 服务 3 列
- 为什么选择 4/8
- 提交新事项 4/8

## Tablet 768–1199px

- 左右 padding 32px
- Hero 5/7 或上下布局
- 服务 2 列
- 为什么选择 1 列 + 3 条列表
- 专业依据 3 列可保留，窄时改 1 列
- 提交新事项上下布局

## Mobile < 768px

- 左右 padding 20px
- Header 简化为 Logo + 菜单
- Hero 上下布局
- 服务单列
- 为什么选择单列
- 专业依据单列
- 代表性工作重点案例在上，次级项目在下
- 提交新事项表单全宽
- 取消所有 pointer-follow 与 sticky
- hover 状态必须有对应 active/focus 状态

# 15. Codex 实现要求

1. 先建立 design tokens，不要在组件内散落硬编码颜色。
2. 每个 section 拆为独立组件。
3. 交互状态使用 CSS class 或组件状态统一管理。
4. Hero 抽象视觉优先使用 SVG + CSS，不使用静态 JPG/PNG。
5. Section reveal 使用 IntersectionObserver。
6. 数字动画只在首次进入 viewport 时触发一次。
7. 所有动画必须支持 reduced motion。
8. 先完成桌面版，再重排移动版，不要简单缩小。
9. 不要自行补充最终营销文案、真实案例名、真实数据或人员信息。
10. 所有占位内容继续使用 placeholder，直到用户提供最终内容。

# 16. 最终视觉原则

> 静态先协调，动态再精致。

> 琥珀金不是第三主色，而是“温度、节点与转化”的稀缺强调色。

> 页面看起来花过设计预算，依靠的是一致的层次、材质、交互反馈和节奏，而不是更多装饰。
