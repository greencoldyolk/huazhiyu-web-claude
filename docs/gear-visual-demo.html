3D 齿轮视觉 · 三种实现方式对比

目标感觉：白色立体齿轮斜置 + 齿面金环光点巡行 + 虚线参考环。三种路线都能做、都能动，取舍不同 —— 页面里三个齿轮都是活的，停留几秒即可观察动画。

推 荐
A · SVG + CSS 3D

方案 A｜纯 SVG + CSS 3D 变换

平面 SVG 齿轮放进 rotateX/rotateZ 倾斜的 3D 平面，内层只做普通 2D 自转 —— 投影后在数学上就等于绕自身轴的 3D 旋转；再用十几层 translateZ 把厚度"叠"出来。

  • 齿轮本体真的在转，齿面金环光点、虚线环、浮动全都可动
  • 全代码 ≈10 KB、零依赖、任意分辨率锐利，改色只动 CSS 变量
  • 与现有 hzy-*-visual 组件同构，可直接替换，SSR 友好
  • 立体感靠分层，特写下侧壁有轻微层纹；高光不如真渲染细腻

✅ 最适合当前站点：品牌感、动效、体积、可维护性的最优平衡。

B · PNG + 叠加动效
图片本体不动 · 动效全在叠加层

方案 B｜PNG/WebP 渲染图 + 动效叠加

齿轮用 AI / 三维软件渲染成图（质感天花板最高），动画发生在叠加层：SVG 金色轨道 + 巡行光点、虚线环流动、光带扫过、整体缓浮。

  • 材质光影最好看，还原渲染稿零损耗、最省事
  • 运行时开销最小，一张 webp + 几个 CSS 动画
  • 齿轮本体永远转不了 —— 透视和光影已"烤"进像素
  • 改色改造型要重新出图；高倍屏放大发虚；轨道需手工对位

适合"图本身已经很美，只需要活起来一点"的场景，可与 A 混用。

C · Three.js 真 3D
未能加载 three.js（此演示需联网获取 CDN 库）。
效果：真实挤出齿轮 + 正交等距相机 + 软阴影，自由旋转。

方案 C｜Three.js 真 3D

把齿轮轮廓 ExtrudeGeometry 挤出成真实几何体，正交相机摆出等距视角，真实光照 + 软阴影落地，想怎么转就怎么转。

  • 最接近渲染图的立体与光影，运动自由度最大
  • 可交互：跟随鼠标微转、滚动视差都容易加
  • 依赖 ~600 KB 库（gzip ≈150 KB），对一个装饰位偏重
  • Angular SSR 下需 afterNextRender 守护，维护成本最高

适合首屏主视觉大场面；tab 区配图用它有点"杀鸡用牛刀"。

落地到 Angular（huazhiyu-web）的建议

  1. 推荐方案 A：把演示里 .iso-plane 的结构搬进 hzy-mechanical-visual —— 你现有的 28s 慢转、金点 20s 巡行、prefers-reduced-motion 全关等规范全部沿用，只是把"平面圆"换成"倾斜平面 + 层叠厚度"。
  2. 核心技巧一句话：外层管姿态（rotate(-20deg) rotateX(38deg) + preserve-3d），内层只做 2D rotate() —— 投影后等价于绕自身轴的真 3D 旋转；厚度用若干层 translateZ 的复制拓出来。
  3. 视角是两个变量：--tiltX 越小齿轮越"立"、越朝向观众，--lean 控制屏幕内对角倾斜的动势 —— A 面板右下角可在「候选 1 · 斜置 / 候选 2 · 平置」之间即时切换对比。齿数、齿深、色带在页面脚本顶部。生产环境可把生成好的 path 直接烤进模板，运行时零 JS。
  4. PNG 并非不能动：让"周边"动（轨道、光点、光带、浮动），整张图就显得是活的 —— 四个领域 tab 若想保留渲染图质感，可统一用 B 的叠加层规范。
  5. 无论哪种：保留 aria-hidden 纯装饰语义 + reduced-motion 降级（本页已内置）。
