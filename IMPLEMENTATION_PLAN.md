# Implementation Plan

## Phase 0 — Repository audit
Inspect the current codebase and reconcile it with the architecture document. No visual implementation.

## Phase 1 — Foundation
Set up tokens, typography, containers, reset, breakpoints, shared motion constants, route rendering strategy, Header/Footer shells.

## Phase 2 — Static responsive homepage
Implement all six sections with placeholders. No signature animation, real API, or file upload.

## Phase 3 — Shared interaction states
Implement button, link, card, form, upload-zone, navigation, reveal, hover, focus-visible, active, disabled, and validation states.

## Phase 4 — Signature visual effects
Implement the Hero layered visual, restrained parallax, evidence material treatment, work-panel treatment, and amber accents.

## Phase 5 — Inquiry form and OSS upload
Implement typed Reactive Forms, validation, direct-to-object-storage upload, progress, retry, success, and error states.

## Phase 6 — QA, accessibility, performance
Test responsive layouts, keyboard navigation, reduced motion, browser compatibility, SSR/SSG output, performance, and production errors.
